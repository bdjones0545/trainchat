Audit all TrainChat user actions for correct role, scope, and agent routing.

Goal:
Make sure every UI action routes to the correct system:
- local exercise edit
- session/day edit
- program/global edit
- architecture/periodization edit
- readiness/check-in flow
- calibration/profile flow
- settings/memory flow
- research/admin flow

Core routing rules:

1. Exercise-level actions
Examples:
- Swap
- Harder
- Easier
- Replace equipment
- Explain exercise

Route:
→ mutation pipeline / direct edit
NOT Performance Architect unless multiple changes require validation.

Required payload:
- source
- actionType
- scope: "exercise"
- programId/systemId
- dayIndex
- exerciseIndex
- exerciseName
- currentExerciseData

2. Session/day-level actions
Examples:
- Make Day 1 harder
- Make Day 1 lighter
- Add finisher to Day 2
- Shorten today’s session

Route:
→ session mutation pipeline
OR Performance Architect if structure needs rebalancing.

Required payload:
- scope: "session"
- dayIndex
- dayName
- active program context

3. Program/global actions
Examples:
- Remove all barbell exercises
- Make this program harder
- Reduce fatigue
- Make this joint-friendly

Route:
→ global mutation pipeline
OR Performance Architect depending depth.

Required payload:
- scope: "program"
- active system
- constraints
- banned/excluded exercises

4. Architecture/periodization actions
Examples:
- Make this more athletic
- Make this better for football
- Progress this for 4 weeks
- Turn this into a 3-day program
- Add speed work across the week
- Deload this week

Route:
→ Performance Architect
→ Program Review / validation
→ Coach response

Required payload:
- scope: "architecture"
- full active program
- goal/sport/context
- research guidance
- profile/memories
- safety constraints

5. Readiness/check-in actions
Examples:
- Check-In
- Adjust today’s plan
- Keep plan as-is

Route:
→ readiness service
→ only mutate plan after explicit user confirmation

6. Calibration/profile actions
Examples:
- Improve My Plan
- Edit training preferences
- Save & Apply
- Use for future programs
- Update current plan

Route:
→ profile/memory update
→ optional Coach/Architect refinement only if user chooses update current plan

7. Settings actions
Examples:
- concise responses
- proactive insights
- auto-adjust recommendations
- memory personalization
- clear coach memory

Route:
→ user settings/profile/memory controls
→ must affect agent prompt behavior where relevant

8. Research/admin actions
Examples:
- analyze research
- approve/reject document
- regenerate chunks

Route:
→ Research Librarian only
Never Coach Agent during user chat.
