We are implementing a PREMIUM “Learn Exercise” modal for TrainChat using the exact component hierarchy and prop structure below.

OBJECTIVE:
When a user taps the learn/play icon on an exercise inside the right panel / Live Program view, open a premium coaching modal that explains:
- why the exercise is in the program
- key coaching cues
- common mistakes
- demo/video access
- optional “ask coach about this” prompts

This must feel like a coaching assist layer, not a generic media popup.

IMPORTANT:
- Keep implementation mobile-first
- Reuse existing styling patterns and dark premium UI
- Do NOT build a new backend API for this
- Use fallback-generated content when structured metadata is missing
- Keep TrainChat as the authority; video links are secondary support

Implement this hierarchy:

==================================================
TOP-LEVEL STATE OWNER
==================================================

The right panel / LiveProgramPanel should own:
- modal open/close state
- selected exercise
- selected context

Add local state like:
- learnModalOpen: boolean
- selectedExercise: LearnExerciseData | null
- selectedContext: LearnExerciseContext | null

Use a centralized handler:

handleOpenLearnExercise(exercise, context)

This prevents each exercise row from owning modal state.

  | "POWER"
