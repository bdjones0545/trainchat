We need to implement HARD CONSTRAINT ENFORCEMENT for program generation.

Right now the system is ignoring explicit user input (e.g., "3 days") and generating incorrect programs (e.g., 4 days).

## GOAL

Ensure that user-stated constraints are ALWAYS respected.

The system must NEVER output a program that violates:
- requested days per week
- requested goal
- requested sport (if provided)

## STEP 1: EXTRACT HARD CONSTRAINTS

Before generation, extract:

constraints = {
  daysPerWeek,
  goal,
  sportFocus,
  sessionDuration (optional),
  equipment (optional)
}

Example:
“I play soccer want a 3 day strength program”

→
daysPerWeek = 3
goal = strength
sportFocus = soccer

## STEP 2: PASS CONSTRAINTS INTO GENERATION

When generating program, explicitly inject:

- MUST be 3 days
- MUST be strength-focused
- SHOULD reflect soccer performance

## STEP 3: ADD VALIDATION LAYER (CRITICAL)

After program is generated, run:

validateProgram(program, constraints)

Validation rules:

IF program.daysPerWeek !== constraints.daysPerWeek
→ FAIL

IF program.goal does not match constraints.goal
→ FAIL

IF constraints.sportFocus exists AND not reflected
→ WARN (not fail, but prefer inclusion)

If validation fails:

- DO NOT render program
- regenerate with stricter instruction

Retry up to 2 times

“I’m fixing your program to match your request…”

- incorrect frequency
