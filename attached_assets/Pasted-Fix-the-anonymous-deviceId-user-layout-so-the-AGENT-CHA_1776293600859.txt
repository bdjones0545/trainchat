Fix the anonymous deviceId user layout so the AGENT / CHAT AREA REMAINS PINNED AS THE PRIMARY TOP-OF-PAGE WORKSPACE during interaction.

OBJECTIVE:
When deviceId / anonymous users type into the agent, submit messages, receive streaming responses, or focus the input, the viewport should remain anchored to the agent/chat workspace at the top of the page.

Right now, anonymous users can drift downward into the conversion-floor content when interacting, which breaks the product hierarchy.

The correct UX is:
- the agent stays visually dominant
- the user remains in the chat/workspace while interacting
- the conversion-floor content below only comes into view when intentionally scrolled to
- input focus, message send, and streamed responses should not push the user down the page

==================================================
PRIMARY RULE
==================================================

For anonymous/deviceId users:

WHILE ACTIVELY USING THE AGENT
- keep viewport anchored to the agent/workspace section
- do not let new messages or layout growth push the user into the lower conversion content
- preserve the feeling that the agent is the main surface

The content below the agent is secondary.
It should never hijack the interaction flow.

==================================================

Audit the anonymous conversion-floor layout and determine why the viewport is not staying at the top agent area.

- autofocus/focus behavior on the input not re-anchoring the viewport
- keyboard opening on mobile shifting the page downward

- flex/overflow/height rules causing the page to resize incorrectly

Audit likely files such as:

- SSE message append handlers
- mobile layout wrappers

Implement a stable top-of-page anchor for the agent/workspace section.

This should feel intentional and stable.

2. user submits a message

4. SSE stream appends content
5. assistant finishes response
6. follow-up input becomes active again

The lower conversion floor should not steal visual priority during any of these states.
