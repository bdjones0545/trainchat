Persist "sport focus" and "exercises to avoid" as first-class user memories.

Goal:
Make sport context and exclusion constraints persist across sessions and be retrieved automatically in future agent calls.

-------------------------------------
PHASE 1 — MEMORY TYPES
-------------------------------------

Add new memory types (or confirm they exist):

- sport_focus
- exercise_exclusion

Update any enum/union:
MemoryType = ... | "sport_focus" | "exercise_exclusion"

-------------------------------------
PHASE 2 — WRITE MEMORIES ON CALIBRATION
-------------------------------------

File:
artifacts/api-server/src/routes/calibrate.ts

When saving calibration:

If sportFocus exists:
- write memory:
  type: "sport_focus"
  value: sportFocus (e.g., "football", "basketball", etc.)
  tags: ["sport", sportFocus]
  priority: high

If exercisesToAvoid exists:
For each exercise:
- write memory:
  type: "exercise_exclusion"
  value: exerciseName
  tags: ["exclude", "exercise"]
  priority: high

Rules:
- de-dupe existing memories (same value/type)
- update timestamp if already exists instead of duplicating
