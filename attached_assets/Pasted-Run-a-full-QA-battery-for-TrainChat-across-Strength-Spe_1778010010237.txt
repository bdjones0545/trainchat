Run a full QA battery for TrainChat across Strength, Speed/Footwork, and Mobility.

Goal:
Find routing, mutation, state sync, Active Programs, versioning, refinement, prescription, copy, and mobile UX issues before UX polish.

Do not change code during this pass.
Test only.
Return a structured report with PASS/FAIL, exact bug, likely file/function, priority, and recommended fix.

Device priority:
- Mobile viewport first
- Desktop second if time allows

Core success criteria:
- Chat is the CEO/main brain.
- Structured UI actions execute deterministically.
- No silent AI fallback.
- No fake “edited” response without DB mutation.
- No stale Live Program Panel.
- No cross-lane Active Programs contamination.
- No unsupported “increase load” language unless actual load data exists.

PHASE 2 — STRENGTH LANE BUILD + EDIT
━━━━━━━━━━━━━━━━━━━━━━

Set focus: Strength.

Build prompt:
“Build me a 3-day strength program for general athletic performance.”

Expected:
- Program builds.
- systemSaved true.
- trainingSystemId returned.
- Active Programs shows Strength active.
- Sidebar/Live Program Panel hydrates with Week 1.
- No duplicate active Strength systems.

Verify program quality:
- 3 sessions.
- Strength-oriented structure.
- Core/accessory work included.
- No unsupported exact load prescriptions unless load values exist.

Test Strength chip:
Click “More Strength.”

Expected:
- Uses direct edit route, not chat/NLP.
- No clarification message in chat.
- Week-level change log created.
- Primary/secondary lifts shift strength-focused.
- Core/accessory exercises do NOT all become 3–6 reps.
- Mobility/prep not converted to strength reps.
- Session identities preserved with suffix/modifier, not replaced.

Test Endurance chip:
Click “More Endurance.”

Expected:
- Direct route.
- Reps/rest visibly shift toward endurance/density.
- Primary lifts move toward moderate reps/rest.
- Accessories/core shift higher-rep or time-based.
- Jumps/power work not turned into sloppy high-rep work.
- Change log says week-level endurance refinement.

Test Explosive chip:
Click “More Explosive.”

Expected:
- Power exercises stay low-rep/high-quality.
- Main lifts may receive speed/intent emphasis.
- Accessories remain supportive.
- Core remains trunk-appropriate.
- Not every exercise gets identical prescription.

Test chat edit:
“Make Day 2 shorter.”

Expected:
- Chat applies session/day edit.
- Only Day 2 changes.
- No full program rebuild.
- systemEdit.applied true.
- Change log scope = session.
- Sidebar updates immediately.

Test exercise swap:
Use Swap on one exercise.
Expected:
- Exercise changes.
- Confirmation badge appears if verified.
- If propagated to future weeks, propagation metadata is full/partial/none.
- Partial propagation warning appears if applicable.

Test misleading copy:
Search visible UI/change logs for:
- increase load
- add weight
- heavier loading
- load progression

Expected:
- Avoided unless actual load values exist.
- Uses “increase difficulty,” “progress intensity,” or “progress resistance when available.”
