We need to harden the ROUTING structured output path in TrainChat.

Context:
The post-migration validation sweep found that the routing/extractProgramJSON path in guestChat.ts is the only structured JSON call in the codebase that does NOT enforce:
response_format: { type: "json_object" }

Right now it relies on:
- prompt instruction telling the model to return pure JSON
- markdown fence stripping regex
- JSON.parse
- temperature: 0

That may work most of the time, but it is unnecessary structural risk, especially since every other structured JSON surface already uses response_format.

Fix the ROUTING path so it is structurally consistent with the rest of the codebase.

TARGET
- guestChat.ts
- specifically the extractProgramJSON routing / structured JSON generation call

REQUIRED CHANGE
Add:
response_format: { type: "json_object" }

to the OpenAI call used for the routing / extractProgramJSON structured output.

Do not redesign prompts.

Do not change business logic.

5. Verify the function still returns the exact same shape expected downstream.
No enum drift.
No renamed keys.
No structural changes to the returned object contract.

6. Validate that this fix does not affect:
- guest chat response generation
