Refine the AdaptiveControlBar to simplify the interface down to ONLY two operating modes:

1. Atlas
2. Check-In

Remove:
- Program
- Readiness

from the top-level control bar entirely.

-----------------------------------
PRODUCT REASONING
-----------------------------------

The current 4-pill version is becoming too dashboard-like and creates unnecessary cognitive load on mobile.

TrainChat works best when the experience feels like:
- an intelligent adaptive coach
- a cinematic coaching OS
- a focused neural interface

NOT:
- a SaaS toolbar
- a utility dashboard
- a multi-module admin panel

The ideal mental model is:

Atlas = talk to the system / coaching intelligence / adaptive programming
Check-In = tell the system how you're feeling today

That’s it.

Program access already exists through the Live Program Panel.
Readiness already exists through the existing modals/check-in flow.

Those do NOT need to be persistent top-level controls.
