We need a broad semantic-intent audit across all three training modes:

- Strength
- Speed / Footwork
- Mobility

Problem:
The system is now strong enough architecturally that the next major risk is semantic misrouting. Certain edit commands are being interpreted too generically or without enough mode awareness.

Example:
“Reduce ground contact time” was treated like a shortening/volume-reduction request instead of a speed/reactivity request.

GOAL
Audit and improve the intent-language system across all three modes so user commands are interpreted according to:
1. the actual target property of the command
2. the active training mode
3. the correct transformation bundle

THIS IS NOT A ONE-PHRASE FIX
We want a mode-aware semantic taxonomy.

AUDIT SCOPE

Audit these layers:
1. intent classification
2. intent-family grouping
3. competing-intent priority rules
4. transformation bundle selection
5. synonym/slang coverage
6. mode-specific interpretation

DO THIS FOR ALL THREE MODES:
- Strength
- Speed / Footwork
- Mobility

KEY PRINCIPLE

Generic verbs like:
- reduce
- increase
- more
- less
- lighter
- heavier
- shorter

must not overpower the actual target property.

The system should prioritize:
TARGET PROPERTY > GENERIC ACTION VERB

Examples:
- “reduce ground contact time” → speed/reactivity
- “lighter on the joints” → mobility or joint-friendly modification
- “more explosive” → power/speed bias depending on mode
- “shorter session” → duration/volume reduction
