Run a universal root-cause audit for why TrainChat main chat commands are not reliably mutating programs.

Do not modify code yet. Audit only.

Problem:
Main chat should be the CEO brain. Any clear training-system command should either:
1. apply a mutation,
2. ask a useful clarification,
3. or clearly say it could not change the program.

It should never:
- lose the original command after a clarification
- classify a scope-only answer like “Day 1” as standalone NO_OP
- say it applied a change when actionType is NO_OP
- route clear edit commands to guidance only

Current confirmed failure:
User: “Make this program harder”
Assistant: asks scope
User: “Day 1”
Actual:
- actionType: NO_OP
- shouldMutate: false
- mutationType: none
- response says it applied a change

This must be fixed for all command families, not just “harder.”

Find the exact point where main-chat commands lose mutation intent or fail to execute.

2. Pending clarification lifecycle
For any command that asks clarification:
Confirm:

If response claims mutation:
Required:

- shouldMutate = true

- persistenceType != none
