We need to restructure the TrainChat exercise library into a decision-ready movement system.

GOAL:
The library should not feel like a random list of exercises.
It should function like a coaching decision engine that supports:
- smart swaps
- pain adjustments
- equipment adjustments
- time compression
- sport-specific programming
- progression / regression logic

The system should reason through:
PATTERN → CATEGORY → EXERCISE → VARIANTS

NOT just:
list of exercise names

--------------------------------------------------
CORE OBJECTIVE
--------------------------------------------------

Convert the current library into a structured movement map the Program Specialist Decision Layer can use reliably.

This means every exercise should be grouped by:
- movement pattern
- role in the session
- difficulty / stress level
- equipment needs
- swap logic

--------------------------------------------------
STEP 3: ADD EXERCISE METADATA (CRITICAL)
--------------------------------------------------

Every exercise entry must include structured metadata.

Use a schema like:

{
  id: string,
  name: string,
  pattern: string,
  category: string,
  role: string,
  equipment: string[],
  difficulty: "low" | "moderate" | "high",
  jointStress: string[],
  neuralDemand: "low" | "moderate" | "high",
  timeCost: "low" | "moderate" | "high",
  sportTransferTags: string[],
  goalsSupported: string[],
  swapOptions: string[],
  regressionOptions: string[],
  progressionOptions: string[],
  notes?: string
}

Example:

{
  id: "back_squat",
  name: "Back Squat",
  pattern: "knee_dominant",
  category: "lower_body",
  role: "primary_strength",
  equipment: ["barbell", "rack"],
  difficulty: "high",
  jointStress: ["knee", "spine"],
  neuralDemand: "high",
  timeCost: "moderate",
  sportTransferTags: ["force_production", "lower_body_strength"],
  goalsSupported: ["strength", "athletic_performance"],
  swapOptions: ["front_squat", "goblet_squat", "safety_bar_squat", "box_squat"],
  regressionOptions: ["goblet_squat", "bodyweight_squat"],
  progressionOptions: ["front_squat", "safety_bar_squat"],
  notes: "Primary bilateral knee-dominant strength option"
}
