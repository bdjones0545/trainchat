Fix the HARDER / EASIER quick-edit behavior on the Your System page so it always resolves to a real, valid exercise change or a real prescription change.

OBJECTIVE:
When a user taps “Harder” or “Easier” on an exercise card (for example Back Squat), TrainChat must never replace the exercise with a generic placeholder like:
- “a harder variation”
- “an easier variation”

That is not acceptable product behavior.

The system must either:
1. resolve to a real named exercise variation from the exercise library
OR
2. apply a real prescription change to the current exercise
OR
3. fail honestly without mutating the displayed exercise

==================================================
PROBLEM
==================================================

Current behavior:
- User taps Harder/Easier
- System updates the card
- Exercise name becomes a generic placeholder like “a harder variation”

This creates a fake-feeling result and breaks trust.

==================================================
CORRECT BEHAVIOR
==================================================

If the user taps Harder on Back Squat, the system should do one of these:

A. REAL EXERCISE SUBSTITUTION
Examples:
- Pause Back Squat
- Tempo Back Squat
- Front Squat
- Safety Bar Squat
- Box Squat (if contextually appropriate)

OR

B. REAL PRESCRIPTION CHANGE
Examples:
- keep Back Squat
- increase sets
- decrease reps / increase intensity bias
- add tempo or pause prescription in notes
- adjust rest

But the system must never commit a generic unresolved label.

==================================================

Audit the full quick-edit pipeline for:
- Harder
- Easier

Trace:
- frontend action handler

I want to identify exactly where the generic label is being created and why it is being committed.

- variation recommendation / substitution logic
- exercise mutation response shaping
- exercise card rendering

==================================================
TASK 2 — REQUIRE REAL EXERCISE RESOLUTION
==================================================

If quick-edit mode chooses substitution:
- the substitute must resolve to a real exercise record or real exercise name from the library
- it must preserve the exercise identity chain cleanly
- it must return the resolved exercise name in the mutation result

Do NOT allow unresolved labels like:
- “a harder variation”
- “an easier variation”
- “harder squat variation”

The replacement must be a real exercise.

==================================================
TASK 3 — ADD PRESCRIPTION-CHANGE FALLBACK
==================================================

If the system cannot confidently resolve a better exercise variation, do NOT write a fake name.

Instead, fallback to modifying the current exercise prescription.

Examples:

For “Harder”:
- keep Back Squat
- +1 set
- slightly lower rep range with higher intensity bias
- add pause/tempo note if supported
- longer rest if needed

For “Easier”:
- keep Back Squat
- reduce sets
- reduce load/intensity bias
- simplify rep target
- adjust rest if needed

This is preferable to a fake substitution.

==================================================
TASK 4 — ADD VALIDATION BEFORE COMMIT
==================================================

Before writing the edit result to DB/UI:

Validate:
- exerciseName is a real named value
- exerciseName is not a placeholder/generic phrase
- mutation result contains either:
  - a resolved replacement exercise
  - or a real prescription modification

If validation fails:
- do not commit the mutation
- return a truthful error
- optionally trigger a retry to resolve a real variation

==================================================
TASK 5 — IMPROVE HARDER / EASIER MAPPING RULES
==================================================

Build explicit variation ladders for common movements.

Example for Back Squat:

Harder options:
- Pause Back Squat
- Tempo Back Squat
- Front Squat
- Safety Bar Squat
- Box Squat (depending on goal)
- heavier lower-rep back squat prescription

Easier options:
- Goblet Squat
- Split Squat
- Belt Squat
- Box Squat (if simplifying mechanics)
- reduced-volume back squat prescription

Do this intelligently by movement pattern and training goal where possible.

At minimum, make squat-family harder/easier logic coherent.
