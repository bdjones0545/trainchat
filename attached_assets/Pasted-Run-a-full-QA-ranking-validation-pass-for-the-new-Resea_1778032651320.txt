1. Run a manual discovery cycle

- candidates stored

- status transitions correct

2. Candidate quality audit

Inspect the discovered candidates.

Verify:
- DOI populated where available
- PubMed IDs valid
- abstract exists
- journal/year/authors normalized
- title dedupe working
- no obvious blog/non-peer-reviewed content
- retracted papers excluded
- category assignment reasonable

Flag:
- duplicate papers
- malformed metadata
- irrelevant papers
- missing abstracts
- broken DOI/source links

Acceptance:
- Candidate queue feels trustworthy and curated.
- Metadata quality is production-ready.

4. Add evidence-type weighting

Improve retrieval scoring hierarchy.

Add scoring boosts:
- meta_analysis: +4
- systematic_review: +3
- consensus_statement / position_stand: +3
- clinical_practice_guideline: +3
- randomized_trial: +2
- cohort_study: +1
- foundational_single_study: +0

If publicationTypes metadata exists:
- use it directly
Otherwise:
- infer conservatively from title/journal metadata.

Acceptance:
- Higher-quality evidence ranks above weaker evidence.
- Meta-analyses dominate retrieval naturally.
