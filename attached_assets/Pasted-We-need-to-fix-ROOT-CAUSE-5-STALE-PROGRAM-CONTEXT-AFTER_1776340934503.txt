- the UI panel shows the updated DB-backed program
- but the SSE handler can still inject currentProgram from old conversation-history JSON
- so the AI reasons from stale program state on the next turn

This causes:
- edits to feel forgotten
- follow-up refinements to act on old structure
- mismatch between what the user sees and what the AI believes
- loss of trust

For refine/edit flows:
DB program truth must override stale conversation-history JSON.
