3. Checkout session route

Create or update:

POST /api/billing/create-checkout-session

Request body:
{
  tier: "starter" | "pro" | "elite",
  billingInterval: "monthly" | "yearly"
}

Backend behavior:
- Requires authenticated user
- Looks up Stripe price by lookup_key:
  trainchat_${tier}_${billingInterval}
- Creates Stripe customer if user does not have stripeCustomerId
- Uses mode: "subscription"
- Adds customer_email if available
- Adds metadata:
  userId
  tier
  billingInterval
  product: "trainchat"
- Adds subscription_data.metadata with same fields
- success_url:
  `${CLIENT_URL}/billing/success?session_id={CHECKOUT_SESSION_ID}`
- cancel_url:
  `${CLIENT_URL}/billing/cancelled`
- Enables Stripe automatic tax only if already configured safely; otherwise leave off
- Returns:
  { url: session.url }

Frontend:
- Pricing modal calls this endpoint.
- Monthly toggle sends monthly.
- Yearly toggle sends yearly.
- Redirects user to Stripe Checkout URL.

4. Webhook route

Create or update:

POST /api/billing/stripe-webhook

Important:
- Must use raw request body for Stripe signature verification.
- Verify with STRIPE_WEBHOOK_SECRET.
- Do not parse JSON before signature verification.

Handle events:
- checkout.session.completed
- customer.subscription.created
- customer.subscription.updated
- customer.subscription.deleted
- invoice.payment_succeeded
- invoice.payment_failed

On checkout.session.completed:
- Read metadata userId, tier, billingInterval
- Store stripeCustomerId
- Store stripeSubscriptionId if available
- Mark user subscription active after successful checkout

On customer.subscription.created/updated:
- Determine active tier from:
  - subscription.metadata.tier
  - price.lookup_key
  - fallback to price ID mapping if needed
- Update user:
  - subscriptionTier: starter | pro | elite
  - subscriptionStatus: active | trialing | past_due | canceled | unpaid
  - billingInterval: monthly | yearly
  - stripeCustomerId
  - stripeSubscriptionId
  - currentPeriodEnd
  - cancelAtPeriodEnd

On customer.subscription.deleted:
- Downgrade user to free or expired tier
- Clear active subscription fields or mark canceled
- Preserve stripeCustomerId

On invoice.payment_succeeded:
- Confirm user subscription remains active
- Update currentPeriodEnd if available
- Trigger/send payment confirmation email receipt if needed

On invoice.payment_failed:
- Mark status past_due or payment_failed
- Do not immediately delete user data
- Optionally show billing warning in app
