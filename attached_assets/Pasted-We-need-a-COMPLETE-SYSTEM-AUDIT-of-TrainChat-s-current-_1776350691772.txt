We need a COMPLETE SYSTEM AUDIT of TrainChat’s current architecture.

OBJECTIVE:
Identify exactly what systems already exist, where logic is duplicated or fragmented, and what is missing — so we STOP rebuilding the same features in different places.

This is NOT a surface-level review. This is a deep architectural inventory.

==================================================
PRIMARY GOALS
==================================================

1. Map ALL existing systems and layers
2. Identify where logic overlaps or duplicates
3. Identify where systems are partially implemented but not unified
4. Identify gaps preventing agent-like behavior
5. Clearly separate:
   - what exists
   - what is incomplete
   - what is missing entirely
