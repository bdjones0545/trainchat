- completion status

1. connects to the agent
2. updates the training system
3. influences the next workout / next sessions / current block

Audit whether this happens now.

==================================================
TASK 5 — VERIFY NEXT-WORKOUT INFLUENCE
==================================================

This is the most important part.

Audit whether the submission influences:
- the next scheduled workout
- the rest of the week
- the current block bias
- exercise substitutions if pain was reported

Expected examples:

CASE A:
Completed + right level + no pain + energized
→ keep progression on track or slightly advance

CASE B:
Completed + too hard + tired/drained
→ reduce next session fatigue / soften progression

CASE C:
Pain reported
→ flag movement family, add monitoring note, reduce risky patterns in next sessions

CASE D:
Skipped / partial
→ agent adapts with recovery logic or reduced progression pressure

==================================================
TASK 6 — MAP EACH FIELD TO SYSTEM BEHAVIOR
==================================================

Create explicit logic for how each field affects the system.

Examples:

completion:
- skipped → recovery bias / progression hold
- partial → partial load carryover / conservative next step
- completed → progression eligible

difficulty:
- too easy → raise challenge
- right level → hold progression
- too hard → reduce load or density

pain:
- mild/moderate/significant/severe
→ trigger pain-aware refinements and movement-family protection

post-session feeling:
- drained / tired → reduce next-session stress
- good / energized → maintain or slightly progress

enjoyment:
- low enjoyment repeatedly → bias toward more preferred training style or exercise variation

notes:
- NLP parse pain location, exercise mentions, substitution requests, constraints
